## Project Overview

Vervet is a desktop MongoDB explorer built with [Wails](https://wails.io) ‚Äî Go for the backend, Vue 3 + Naive UI for the frontend. The Wails framework embeds the compiled frontend into the Go binary and bridges Go methods to TypeScript via auto-generated bindings.

## Commands

### Run in development mode (recommended)
```bash
wails dev
```
This starts the Go backend and the Vite dev server together, with hot-reload.

### Build production binary
```bash
wails build
```

### Frontend only (from `frontend/`)
```bash
bun run dev       # Vite dev server
bun run build     # Type-check + production build
bun run lint      # ESLint + vue-tsc --noEmit
bun run format    # Prettier
bun run test      # Vitest (unit tests)
```

### Backend (Go)
```bash
go test ./...                        # All Go tests
go test ./internal/servers/...       # Single package
go test -run TestName ./internal/... # Single test
```

## Architecture

### Wails binding pattern

Go methods are exposed to the frontend through four **Proxy** structs in `internal/api/`:
- `ServersProxy` ‚Äî CRUD for registered servers
- `ConnectionsProxy` ‚Äî connect/disconnect/query MongoDB
- `SystemProxy` ‚Äî system info, fonts
- `SettingsProxy` ‚Äî app settings

All proxy methods return `Result[T]` or `EmptyResult` (defined in `internal/api/result.go`) instead of Go errors, so the frontend can check `result.isSuccess` without handling thrown exceptions.

Wails generates TypeScript bindings into `frontend/wailsjs/go/` from these proxies. **Do not edit files in `frontend/wailsjs/` manually** ‚Äî they are regenerated by `wails dev`/`wails build`.

### Backend packages (`internal/`)

| Package | Responsibility |
|---|---|
| `api/` | Wails-bound proxies; error ‚Üí Result conversion |
| `app/` | App lifecycle; wires all managers together |
| `servers/` | CRUD for registered server metadata |
| `connections/` | Active MongoDB connections (mongo-driver) |
| `connectionStrings/` | Securely stores/retrieves MongoDB URIs via OS keyring (`go-keyring`) |
| `infrastructure/` | Generic YAML file store; reads/writes to `~/.config/vervet/` |
| `settings/` | App settings persistence |
| `models/` | Shared Go types |
| `system/` | OS info, font detection |

Connection URIs are **never stored in the YAML config** ‚Äî they go through the OS keyring. Server metadata (name, ID, parentID, colour) is stored in `~/.config/vervet/connections.yaml`.

### Frontend structure (`frontend/src/`)

| Path | Purpose |
|---|---|
| `app/` | `App.vue` (root: theme, i18n, global dialogs), `AppContent.vue` |
| `features/server-pane/` | Server tree UI, `serverStore.ts` (Pinia), ServerDialog, GroupDialog |
| `features/data-browser/` | `DataBrowserTree.vue`, `browserStore.ts` (connections + db/collection tree) |
| `features/tabs/` | Tab management; `NavType`: `Servers` \| `Browser` |
| `features/settings/` | Settings dialog and `settingsStore.ts` |
| `stores/` | Shared `useDialogStore` (controls visibility of all modal dialogs) |
| `i18n/en-GB/` | All UI strings |
| `utils/` | Theme, logging, dialog helpers |

### Data flow

1. Vue component ‚Üí Pinia store action
2. Store action calls auto-generated wailsjs function (e.g., `connectionsProxy.Connect(serverId)`)
3. Wails routes call to Go proxy ‚Üí manager ‚Üí returns `Result[T]`
4. Store checks `result.isSuccess`, updates state or shows error via `useNotifier()`

### Frontend path aliases
- `@` ‚Üí `frontend/src/`
- `wailsjs` ‚Üí `frontend/wailsjs/`

### Auto-imports
- Naive UI components are auto-imported via `unplugin-vue-components` (no explicit imports needed in `<script>`)
- `useDialog`, `useMessage`, `useNotification`, `useLoadingBar` are auto-imported via `unplugin-auto-import`
- Icons come from `@vicons/tabler` via `unplugin-icons` (import as `import IconName from '~icons/tabler/icon-name'`)

### Wails runtime events
The Go backend emits events that the frontend can listen to:
- `"connection-connected"` ‚Äî emitted with `serverID` when a connection is established
- `"connection-disconnected"` ‚Äî emitted with `serverID` on disconnect
